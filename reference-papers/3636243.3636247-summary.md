# More Robots are Coming: Large Multimodal Models (ChatGPT) can Solve Visually Diverse Images of Parsons Problems

## Summary

This paper tests whether **large multimodal models** can solve **visual Parsons problems**, which are programming puzzles shown as images of shuffled code blocks. The main conclusion is clear: **GPT-4V was extremely strong**, and this weakens the idea that switching to visual programming problems will protect assessments from AI misuse. 

## What the paper studies

The authors compare:

* **GPT-4V**
* **Google Bard** using Google Lens-style text extraction

They use **6 Python Parsons problems**, each shown in **4 different visual formats** drawn from common Parsons platforms:

* JS-Parsons
* Runestone
* Epplets
* EvoParsons

Each problem-format pair was tested **5 times per model**, for **120 responses per model** and **240 total**. 

## Research questions

The paper asks two things:

1. **How well can multimodal models solve Parsons problems from images?**
2. **Do different visual layouts affect performance?** 

## Method

The authors created a dataset of screenshot-style Parsons problems with varied fonts, colors, layouts, and UI clutter. They then used the same structured prompt for both models:

* extract the text from the image
* identify the task and code blocks
* reorder and indent the blocks to solve the problem

They did not allow follow-up clarification. Only the **first answer** counted. 

## Main findings

### 1. GPT-4V performed very well

GPT-4V solved **116 out of 120** attempts, or **96.7%**. It was almost perfect across all visual formats. Its only errors came from one problem where it slightly rewrote a code block instead of copying it exactly. 

### 2. Bard was much weaker

Bard solved **69.2%** of attempts. It often failed because it:

* refused to answer
* hallucinated or rewrote code blocks
* extracted text incorrectly
* produced wrong sequencing

The paper notes a **7.5% refusal rate** for Bard. 

### 3. Visual complexity mattered more for Bard than GPT-4V

GPT-4V handled different visual styles well. Bard struggled more with cluttered formats, especially **Epplets**, which had smaller fonts, more windows, more interface text, timers, and other distracting elements. 

### 4. Prompting mattered a lot

A simple prompt like “solve the Parsons problem in the image” gave performance closer to **50%** for GPT-4V during prompt engineering. A more explicit prompt improved results substantially. The paper argues that **model choice and prompt design strongly affect outcomes**. 

## Why this matters

The paper’s real point is about **assessment**.

Some educators hoped that turning coding questions into images or visual puzzles would make it harder for students to use AI. This paper shows that this protection is already weak, at least for strong multimodal models. GPT-4V can solve visual Parsons problems at a very high rate, so **visual formatting alone is not a reliable defense**. 

The authors suggest two broad responses:

* try adversarial or visually confusing formats, though this may also hurt students
* rethink assessment more fundamentally, using alternative forms of evaluation rather than relying on traditional task formats 

## Limitations

The paper is useful, but narrow.

Main limits:

* only **2 models** were tested
* only **6 Parsons problems**
* only **4 visual representations**
* only **Python**
* it focuses on **Parsons problems**, not all visual programming tasks like UML, tables, graphs, or trees 

## Bottom line

This paper shows that **modern multimodal models, especially GPT-4V, can solve image-based Parsons problems very reliably**. That means using visual programming questions is **not enough** to keep assessments AI-resistant anymore. 

## One-sentence version

**The paper finds that GPT-4V solved 96.7% of visually diverse Parsons problems, showing that image-based programming assessments are no longer a strong barrier to AI use.**
